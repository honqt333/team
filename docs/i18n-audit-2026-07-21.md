## 6. Existing tooling

`php artisan translations:check` (in `app/Console/Commands/TranslationsCheck.php`)
**detects missing keys** but does **not detect placeholder values**
like `'[AR] work_orders.actions.cancel'`. A guard test exists
(`tests/Feature/I18nCoverageTest.php`, added in commit `441eee89`) that
asserts the missing-key count is 0 — that's why the command reports a
clean bill of health even though the user-facing layer is broken.

## 7. Recommended next steps (in priority order)

1. **Add `lang/ar/pagination.php` and `lang/ar/passwords.php`** — small
   surface area, every user hits pagination, and passwords affects the
   "we emailed your reset link" flow.
2. **Translate the 23 zero-percent files** — many are short (1–33
   keys) and high-traffic (nav, users, customers, dashboard, login).
3. **Close the gap in the 9 partially-translated files** — 1,599
   placeholder keys remain, mostly in `work_orders`, `hr`,
   `inventory`, `quotes`, `company_profile`, `invoices`.
4. **Extend `TranslationsCheck` to flag placeholder values** — change
   the `translations:check` command to also report any value matching
   `/^\[AR\] /` and fail `--strict` on those. This is the cheap,
   load-bearing guard.
5. **Migrate hardcoded Vue text** — for the 52 files with no `$t()` at
   all, the work is non-trivial but well-scoped. Start with the
   `System/*` admin pages (high-traffic) and the print templates.
6. **Migrate hardcoded PHP flash messages** — `with('success', '...')`
   and `with('error', '...')` strings should become `with('success',
   __('key'))` and the keys land in the existing `messages.php` /
   feature-specific files.
