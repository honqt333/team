# 09_DEPLOYMENT_RULES: قواعد وبروتوكولات النشر والتشغيل (Deployment)

تحدد هذه الوثيقة الخطوات والمعايير الواجب اتباعها لنشر وتحديث الكود البرمجي في البيئة الإنتاجية (Production Environment) لضمان عدم توقف خدمات المراكز أو فقدان البيانات.

---

## 1. خطوات النشر القياسية (Deployment Steps Workflow)

تتم عملية نشر التحديثات البرمجية باتباع تسلسل الخطوات التالية بدقة:

1. **جلب الكود الجديد**: جلب النسخة المستقرة المعتمدة من المستودع الرئيسي:
   ```bash
   git pull origin main
   ```
2. **تثبيت الاعتماديات (Dependencies)**: تثبيت حزم المخدم دون الحزم التطويرية مع تحسين التحميل التلقائي:
   ```bash
   composer install --no-dev --optimize-autoloader
   ```
3. **تنصيب حزم الواجهات**: تحميل وتحديث حزم npm وتجهيز ملفات العرض الثابتة (Production Build):
   ```bash
   npm ci
   npm run build
   ```
4. **تحديث قاعدة البيانات**: تشغيل الهجرات الجديدة بأمان (راجع بروتوكول الهجرة الآمن):
   ```bash
   php artisan migrate --force
   ```
5. **تهيئة وتحسين الذاكرة المؤقتة (Optimize Cache)**: مسح وإعادة بناء الكاش لضمان كفاءة السرعة (راجع بروتوكول الكاش).

---

## 2. بروتوكول تهيئة الكاش (Cache Optimization Protocol)

في البيئة الإنتاجية، يجب دائماً تفعيل تخزين إعدادات ومسارات وجداول الاستعلامات في الذاكرة المؤقتة لزيادة السرعة، مع تنظيفها عند كل عملية نشر جديدة لضمان عمل التحديثات:

* **كاش الإعدادات**:
  ```bash
  php artisan config:cache
  ```
* **كاش المسارات (Routes)**:
  ```bash
  php artisan route:cache
  ```
* **كاش واجهات العرض (Views)**:
  ```bash
  php artisan view:cache
  ```
* **كاش الأحداث والمهام الجدولية**:
  ```bash
  php artisan event:cache
  ```

---

## 3. بروتوكول تحديث قاعدة البيانات في الإنتاج (Production Migration Safety)

لتحديث الجداول دون التسبب في توقف النظام (Downtime) أو تعطل العمليات المالية الجارية:

1. **أخذ نسخة احتياطية (Database Backup)**: يجب عمل نسخة احتياطية كاملة من قاعدة البيانات قبل تشغيل أي هجرة تعديل هيكلية تحتوي على تغييرات في الجداول الكبرى.
2. **التشغيل القسري الآمن**: استخدم خيار التجاوز القسري لتجاوز التأكيد اليدوي في بيئة الإنتاج:
   ```bash
   php artisan migrate --force
   ```
3. **تجنب العمليات الحصرية للمفاتيح (Exclusive Lockouts)**: يُمنع تشغيل تعديلات تتطلب وقتاً طويلاً لإعادة فهرسة الحقول (indexing) أو تغيير الأنواع (types) في أوقات ذروة عمل مراكز الصيانة؛ يجب جدولتها وتحديثها في أوقات الإغلاق.

---

## 4. التحقق من متغيرات البيئة (Environment Checks)

قبل إعلان نجاح عملية النشر، تأكد من مطابقة الإعدادات التالية في ملف `.env` الإنتاجي:
* وضع العمل نشط في الإنتاج: `APP_ENV=production`.
* إيقاف وضع استكشاف الأخطاء: `APP_DEBUG=false`.
* تحديد مفتاح التشفير الآمن: `APP_KEY` (مكتمل ومولد بنجاح).
* تفعيل التخزين الموزع للكاش والطوابير في حال توسع الخوادم.
* التحقق من سلامة الربط بمزودي بوابات الدفع (Moyasar, Tap, Tamara) وتغير عناوين الاستدعاء (Base URLs) وعناوين الاستجابة (Callback URLs) للبيئة الحية.
