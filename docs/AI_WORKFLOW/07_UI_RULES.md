# 07_UI_RULES: معايير تصميم واجهات المستخدم (UI/UX)

تحدد هذه الوثيقة المبادئ التوجيهية لتصميم وتطوير واجهات المستخدم ثنائية اللغة في نظام **Khidmah Pro** لضمان التناسق البصري وسلاسة الاستخدام على كافة الشاشات.

---

## 1. الهوية اللونية للموديولات (Module Branding)

يتم تخصيص لون أساسي لكل موديول في النظام لتسهيل التعرف الذهني البصري للمستخدم على موقعه داخل لوحة التحكم:

| الموديول التشغيلي | اللون الأساسي | الكلاس البرمجي (Tailwind) |
| :--- | :--- | :--- |
| **كروت العمل (Work Orders)** | Indigo | `indigo-600` |
| **الفواتير والمالية (Invoices)** | Emerald / Green | `emerald-600` |
| **العملاء (Customers)** | Cyan | `cyan-600` |
| **المركبات (Vehicles)** | Blue | `blue-600` |
| **عروض الأسعار (Quotes)** | Amber | `amber-500` |
| **الإعدادات العامة (Settings)** | Gray | `gray-700` |

---

## 2. الخطوط والأبعاد (Typography & Spacing)

* **الخطوط المعتمدة**:
  * اللغة العربية: خط **Tajawal** أو **Noto Kufi Arabic**.
  * اللغة الإنجليزية: خط **Inter** أو **Outfit**.
* **مقياس حجم النصوص**:
  * العناوين الكبرى: `text-3xl` إلى `text-4xl` مع وزن عريض `font-black`.
  * العناوين الفرعية: `text-lg` إلى `text-xl` مع وزن `font-bold`.
  * النصوص العادية: `text-sm` أو `text-base` مع وزن `font-normal`.
  * حقول الإدخال والملصقات: `text-xs` مع وزن عريض `font-bold` للحروف الكبيرة.
* **الهوامش والأبعاد (Spacing)**:
  * الفراغات الداخلية للبطاقات الكبرى: `p-4 sm:p-6`.
  * التباعد بين الحقول: `space-y-6` أو `gap-4`.

---

## 3. معايير دعم الشاشات والجوال (Responsiveness & Mobile-First)

يجب تصميم كافة العناصر لتكون متوافقة مع شاشات الجوال كأولوية قصوى:

* **ترتيب العناصر**: استخدام التنسيق المرن `flex-col sm:flex-row` لضمان ترتيب المحتوى عمودياً على الموبايل وأفقياً على الشاشات الكبيرة.
* **الأزرار**: يجب أن تمتد الأزرار لتأخذ العرض الكامل للشاشة `w-full` في شاشات الجوال، وتتحول لعرض تلقائي `sm:w-auto` في الشاشات الكبيرة.
* **الجداول**: يجب وضع كافة الجداول داخل حاوية تمرير أفقي مخصصة لضمان عدم خروج البيانات عن حدود شاشة الجوال:
  ```vue
  <div class="overflow-x-auto">
      <table>...</table>
  </div>
  ```

## 4. الوضع الداكن ودعم اللغات (Dark Mode & Localization RTL/LTR)

### أ. دعم الوضع الداكن (Dark Mode)
يجب توفير بديل لوني لكل عنصر في الواجهة باستخدام البادئة `dark:`:
* الخلفيات: `bg-white` للوضع الفاتح و `dark:bg-gray-800` أو `dark:bg-gray-900` للوضع الداكن.
* النصوص: `text-gray-900` للوضع الفاتح و `dark:text-gray-100` للوضع الداكن.
* الحدود: `border-gray-200` للوضع الفاتح و `dark:border-gray-700` للوضع الداكن.

### ب. دعم الاتجاهين واللغات (RTL & LTR)
* **منع الاتجاهات الصلبة**: يُحظر استخدام اتجاهات ثابتة مثل `left-0` أو `right-0`. بدلاً منها، استخدم الاتجاهات المنطقية المتوافقة مع اتجاه الصفحة تلقائياً مثل `start-0` و `end-0`.
* **دوران الأيقونات**: يجب عكس اتجاه أيقونات الأسهم والتنقل تلقائياً عند تغيير اللغة باستخدام الكلاس `rtl:rotate-180`.
* **الترجمات**: لا تكتب أي نصوص مباشرة (Hardcoded) في ملفات الواجهة. استخدم دائماً `$t('module.section.key')` واستخرج الكلمات لملفات `ar.json` و `en.json`.
* **الأرقام**: تُعرض الأسعار والأعداد والنسب دائماً بالأرقام الإنجليزية (0-9) لضمان القراءة البرمجية والوضوح المحاسبي العالمي.

## 5. المكونات البصرية الموحدة (Standard UI Elements)

* **النماذج المنبثقة (Modals)**: تُبنى استناداً إلى `BaseModal.vue` وتأتي بأحجام معيارية (`sm` للتأكيد، `md` للنماذج البسيطة، `lg` للنماذج المتوسطة، `xl` للنماذج ذات التبويبات).
* **لوحة السيارة السعودية**: لعرض لوحة السيارة، استخدم دائماً المكون الموحد `SaudiPlateDisplay.vue` ولإدخال اللوحة استخدم `SaudiPlateInput.vue`.
* **توقيع العميل**: التوقيع الرقمي يتم التقاطه وتخزينه حصرياً باستخدام المكون المشترك `SignaturePad.vue`.
